Blueprint Proyek Sistem E-Voting Lokal

Sistem Informasi Pemilihan Umum Lokal Berbasis Android dan Web Server

Tujuan Proyek

Modernisasi

Menggantikan proses manual dengan sistem digital untuk efisiensi.

Akurasi

Mengurangi potensi kesalahan perhitungan suara manusia.

Transparansi

Menyediakan monitoring perolehan suara secara real-time di TPS.

Keamanan

Menjamin kerahasiaan pilihan dalam jaringan lokal yang terisolasi.

Arsitektur Sistem

Sistem berjalan pada jaringan lokal (Intranet) yang terisolasi di setiap TPS.

Alur Kerja Sistem

  1. Petugas KPPS menyalakan Server Lokal dan Access Point di TPS.
  2. Admin mengimpor data DPT dan data Kandidat ke server.
  3. Sistem menghasilkan QR Code untuk koneksi Wi-Fi, lalu dicetak.
  4. Petugas melakukan uji coba fungsi sistem.

  1. Pemilih diverifikasi manual, lalu menerima username & password.
  2. Pemilih memindai QR Code untuk terhubung ke Wi-Fi lokal. Aplikasi memvalidasi koneksi.
  3. Pemilih login ke aplikasi menggunakan kredensial yang diberikan.
  4. Halaman pemilihan kandidat tampil, pemilih membuat pilihan dan konfirmasi.
  5. Aplikasi mengirim data (ID pemilih & ID kandidat) ke server.
  6. Proses Kunci di Server: Server memvalidasi pemilih, lalu secara terpisah (1) Mengubah status pemilih menjadi 'SUDAH', dan (2) Menambah +1 suara ke kandidat yang dipilih. Tidak ada jejak yang menghubungkan keduanya.
  7. Aplikasi menerima konfirmasi sukses dan otomatis logout.
  8. Layar monitoring secara real-time menampilkan nama pemilih yang sudah memilih dan memperbarui grafik perolehan suara.

Teknologi yang Digunakan

Komponen Teknologi Keterangan
Aplikasi Klien Android (Kotlin/Java) Aplikasi mobile untuk pemilih.
Backend (API) Node.js (Express.js) Mengelola logika bisnis dan koneksi database.
Frontend Monitoring HTML, CSS, JavaScript Halaman web dengan WebSocket untuk update real-time.
Database SQLite / MySQL Menyimpan data pemilih, kandidat, dan agregat suara.
Web Server Nginx / Apache Menjalankan aplikasi backend dan frontend.

Aspek Keamanan

  • Isolasi Jaringan: Sistem berjalan di jaringan lokal tertutup tanpa akses internet.
  • Aplikasi Terproteksi: Aplikasi hanya berfungsi jika terhubung ke Wi-Fi yang benar.
  • Otentikasi Sekali Pakai: Satu akun hanya bisa digunakan untuk memilih satu kali.
  • Kerahasiaan Suara: Desain database memisahkan data pemilih dan data pilihan untuk menjaga anonimitas.
  • Keamanan Fisik: Perangkat keras (server, HP) diawasi langsung oleh petugas KPPS.
  • Enkripsi Password: Password pemilih disimpan dalam bentuk hash di database.

Rencana Implementasi (Timeline)

Fase 1: Pengembangan (8 Minggu)

Pembuatan Aplikasi Android, Backend API, Database, dan Web Monitoring.

Fase 2: Pengujian Internal (4 Minggu)

Uji fungsionalitas, uji beban (stress test), dan audit keamanan internal.

Fase 3: Simulasi & Pelatihan (2 Minggu)

Simulasi pemilihan di lingkungan terkontrol dan pelatihan untuk petugas KPPS.

Fase 4: Implementasi Lapangan (Hari-H)

Deployment sistem di seluruh TPS yang ditunjuk untuk hari pemilihan.

Fase 5: Audit & Pelaporan (Pasca-Pemilu)

Ekstraksi hasil perhitungan dari setiap server lokal dan audit sistem.
